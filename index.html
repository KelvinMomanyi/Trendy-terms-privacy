Technical Architecture

Component Hierarchy

  1. TrendySlider (Main Component)
    • Orchestrates product discovery experience
    • Manages state for carousel, filters, and AI suggestions
    • Coordinates personalization pipeline
  2. ProductGrid
    • Displays products in immersive 3D grid layout
    • Handles touch/pointer interactions with momentum scrolling
    • Shows product detail modals
  3. TrendyFilters
    • Category, price, and time-based filtering
    • Drawer interface with swipe gestures
  4. AISuggestionCard
    • Displays AI-generated product recommendations
    • Triggered by cart additions or favorites

Product Pipeline (6-Stage Process)

Stage Description Purpose
1. Raw Products Combine recommended & popular products from Shopify Initial data aggregation
2. Personalization Apply interest-based scoring using user preferences Relevance ranking
3. Gender Filtering Filter by gender affinity (MALE/FEMALE/NEUTRAL) Demographic targeting
4. Social Proof Add engagement metrics (saves, momentum, trending) Social validation
5. User Filters Apply category, price, and time filters User refinement
6. Final Output Top 20 products with fallback logic Optimized feed

Data Collection & Processing

Information Collected

1. Shopify Buyer Attributes

  • Gender Affinity: MALE, FEMALE, or NEUTRAL classification
  • Category Affinities: User's interest in product categories with ancestor hierarchies
  • Source: Provided by Shopify's useBuyerAttributes() hook

2. Behavioral Tracking

  • Product Swipes: Tracks which products users view in carousel
  • Add to Cart: Records cart additions with timestamps
  • Favorites/Saves: Monitors saved products
  • Product Views: Logs product detail page visits

3. User Preferences (Stored Locally)

  • Interests: Array of category preferences
  • Liked Products: IDs of favorited items
  • Disliked Products: Products to exclude from recommendations

4. Shopping Cart Data

  • Product ID and variant ID
  • Product title and price
  • Quantity
  • Product type and tags
Important: All behavioral tracking is processed through the custom personalizationService and stored locally in the user's browser. No personal identification data is collected.

Personalization Engine

How Personalization Works

1. Interest Mapping

The app combines multiple data sources to understand user preferences:

  • User's explicitly tracked interests from behavior
  • Shopify's category affinity data
  • Product category ancestor hierarchies

2. Product Scoring Algorithm

Products are scored based on:

  • Category Match: Products in user's interest categories receive higher scores
  • Behavioral Signals: Previous likes, cart additions, and views influence ranking
  • Dislike Filtering: Removes products user has explicitly disliked
  • Gender Alignment: Filters products based on gender affinity when available

3. AI Recommendation Service

The aiRecommendationService provides:

  • Complementary Products: Suggests items that pair well with viewed/favorited products
  • Upsell Suggestions: Recommends higher-value or related items based on cart contents
  • Context-Aware Messaging: Generates personalized recommendation copy
Privacy First: All AI processing happens using aggregated, anonymized behavioral patterns. Individual user identity is never part of the recommendation algorithm.

Privacy & Security

Data Protection Principles

1. No Personal Identification

TRENDY does not collect or store:

  • Names, email addresses, or contact information
  • Payment information (handled by Shopify)
  • Location data beyond what Shopify provides
  • Device identifiers or IP addresses

2. Local-First Storage

User preferences and behavioral data are stored locally in the browser using:

  • React state management (session-based)
  • No server-side user profile databases
  • Data persists only during the shopping session

3. Shopify Integration Security

All e-commerce transactions leverage Shopify's secure infrastructure:

  • PCI-compliant payment processing
  • Encrypted checkout process
  • Shopify's privacy and data handling policies apply

4. Third-Party Data Sharing

TRENDY shares data only with:

  • Shopify: For cart operations and product data
  • No other third parties: No advertising networks, analytics services (beyond Shopify's), or data brokers

User Rights

  • Right to Access: Users can view their behavioral patterns through the app interface
  • Right to Deletion: Clear browser data to remove all locally stored preferences
  • Right to Opt-Out: Disable personalization features through filter settings
